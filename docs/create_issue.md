# Issue作成ガイドライン
このドキュメントは、開発者やAIが迷わず実装できる高品質なIssueを作成するためのガイドラインです。

## テンプレート構造

### 1. タイトル
`[種別] 簡潔な説明`

種別: `Feature`, `Bug`, `Refactor`, `Docs`, `Test`, `Chore`

### 2. 概要
2-3文で実装内容を説明

### 3. 背景・目的
なぜ必要かを簡潔に記述

### 4. 詳細仕様

**機能要件**
- 具体的な動作を箇条書き
- 入力条件、出力結果を明記

**技術仕様**
- APIエンドポイント
- データ構造（JSON例）
- 使用ライブラリ
- ファイル配置

**UI/UX仕様**（該当時）
- 使用コンポーネント
- ローディング/エラー表示
- 操作フロー

### 5. 実装範囲
**含まれるもの**
- [ ] 実装項目1
- [ ] テスト追加
- [ ] ドキュメント更新

**含まれないもの**
- 今回対象外の機能

### 6. 受け入れ条件
- [ ] 動作確認項目を具体的に列挙
- [ ] テストがパスする
- [ ] コードレビュー完了

### 7. 技術的考慮事項
- パフォーマンス、セキュリティ、エラーハンドリング等

### 8. 関連リソース
- 関連Issue: #xxx
- 参考実装、API仕様書へのリンク

## バグ報告時の追加項目
```markdown
## 再現手順
1. 具体的な操作手順

## 期待される動作
あるべき動作

## 実際の動作
発生している問題

## 環境
- OS、ブラウザ、バージョン

## エラーログ
該当箇所を添付
```

## AI実装時の追記
```markdown
## AI実装ガイド
- コーディング規約: `docs/coding-standards.md`
- 参考実装: `src/components/xxx`
- 禁止事項: xxx
- テストカバレッジ: 80%以上
```

## 良いIssueの条件
1. すぐ作業開始できる具体性
2. 完了判断が明確
3. 必要な技術情報が揃っている
4. 依存関係が明記されている
